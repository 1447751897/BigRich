# 迭代:页面美化 / 体验升级

## 目标
对**已交付首版**的 Web 前端做**视觉与体验升级**:更精致、有"大富翁"氛围的自创主题皮肤;棋盘**桌面 + 手机竖屏都一屏自适应、无需滚动**;玩家能**直观看到自己/各玩家的位置**(棋子高亮 + 移动动画);**掷骰动画**;**音效**。**不改玩法、不改后端协议与数值。**

## 与项目 baseline 的关系
- 基于首版 baseline(`project/01–05` 均 done,main 38 测试全绿、可一键试玩)。
- **本迭代只动渲染/交互层(`public/` 前端)**,不碰规则引擎(`src/engine/`)与网关协议(`src/gateway/`)——沿用 baseline 的"端无关 + 服务端权威"架构(`02-design §1/§5.1`)。
- 沿用 baseline 硬约束:**去 IP 化**(自创主题,规避 Monopoly 商标元素)。

## 本次侧重与裁剪
- **侧重**:`01-requirements`(本文已澄清落地)→ `02-design`(UI/视觉/交互/响应式/音效方案,产品+设计/架构)→ `04-mvp`(前端实现)→ `05-acceptance`(一屏自适应/位置可见/动画/音效/不回归玩法的验收)。
- **`03-tech-stack`**:大概率为**增量选型**(是否引入轻量 UI/动画/音频库,如 CSS 动画 / canvas / howler 等),由架构师按需补;若无新选型可标 `n/a`。

## 收尾回流要点(归档时填)
- 自创主题与响应式/动画/音效方案中"长期成立"的结论(如最终主题、响应式断点策略、音效开关约定)归档时回流 baseline `02-design`/`03-tech-stack`;一次性素材细节留在本迭代。
