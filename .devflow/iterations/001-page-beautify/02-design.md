# 方案设计 — 迭代:页面美化

## 1. 总体思路
(UI/视觉/交互升级的总体方案与关键取舍;强调只动渲染层、沿用端无关+服务端权威。)

## 2. 视觉/主题方案
**2026-05-30 定稿方向(reporter 拍板):正统大富翁(品类)质感 + 克制配色。**
- **桌台/盘面**:深绿桌台(felt)+ 奶油盘面 + 深描边;中心为奶油中心区 + 对角自创品牌字「环游都市」。
- **地产格**:经典样式 = 格顶纯色色带 + 名称 + 价格;低饱和经典色组(收敛掉原多彩高饱和的"花哨")。
- **配色 token**:`--felt`(深绿桌台)、`--board/--paper`(奶油)、`--ink`(炭黑)、点缀 `--accent`(低饱和经典红)/`--accent2`(经典绿)/`--gold`;街区/棋子色板统一降饱和。
- **字体气质**:展示字体(`--display`,粗体 + 字距)用于品牌/标题/中心字,正文用系统无衬线。
- **去 IP 复核**:仅借鉴品类视觉语言(奶油盘面/色带/方形棋盘),**命名与文案全自创**(环游都市/翡翠湾等),不出现受保护商标/标志,不照搬经典色序原样。若 reporter 要求命名也贴近经典 → 需先确认是否放宽去 IP 硬线(见 §7)。

## 3. 响应式 / 一屏布局
桌面与手机竖屏的一屏自适应方案(棋盘缩放/重排、操作区在小屏的收纳)。

## 4. 关键交互/动画
棋子定位与移动动画、掷骰动画(结果以服务端事件为准)、成交/破产等反馈。

## 5. 音效方案
音效清单、触发点、静音/音量、素材来源与加载。

## 6. 备选方案与取舍
（CSS 动画 vs canvas、是否引入 UI/动画/音频库等。）

## 7. 风险与未决
（性能/低端机降级、素材版权、与既有事件流的对接点。）
