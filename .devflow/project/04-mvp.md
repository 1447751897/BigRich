# MVP 范围与计划

> 输入:`02-design.md`(done)+ `03-tech-stack.md`(D-001~D-005)+ `05-acceptance.md`(§4B 回归 seed)。
> 本轮按架构师交底"最高风险件优先",先打通**回合主循环 + 交易/拍卖/抵押三子状态机**,并补齐回归测试。

## 0. 技术选型拍板(开发工程师补 03-tech-stack 的 proposed/待定)
| 领域 | 结论 | 说明 |
|------|------|------|
| 语言/运行时 | **TypeScript + Node.js (≥20)** | 端无关核心:同一份纯 TS 引擎可被 Web/H5 与微信小程序复用;长连接/内存态友好 |
| 实时传输 (D-002) | **WebSocket(`ws`)** accepted | 双向低延迟;小程序端后续写传输适配,协议层端无关 |
| 规则引擎形态 (D-003) | **纯逻辑库** `reduce(state,cmd)->{state,events}` | 不依赖传输/渲染/系统时钟,可单测 |
| 并发模型 (D-001) | **单房间单写者**(有序命令队列串行) | 出价原子性天然成立 |
| 存储 (D-005) | **内存主存 + JSON 快照** accepted | 每次权威变更后落快照 seam(`onSnapshot`);首版内存即权威态,落盘介质用本地 JSON 起步 |
| 部署 | 单实例起步,按房间分片粘性路由 | 熟人低并发,放量转 A 再评估 |

## 1. MVP 范围(本轮 P0)
- ✅ **端无关规则引擎核心**:服务端 RNG 掷骰、移动、落点结算(买地/租金/税/机会命运/监狱/破产)、净资产结算排名。
- ✅ **回合主循环 FSM**:轮转、回合上限/掉线兜底、超时合成默认命令。
- ✅ **§4B 三子状态机**:
  - 交易(仅自回合发起、20s 响应、还价仅 1 次、超时/掉线自动拒绝、原子过户资金守恒);
  - 拍卖(全员公开、每轮 10s、出价原子性防同价/超额、连续一轮无人加价成交/全员放弃流拍、掉线直接放弃);
  - 抵押/赎回(仅自回合、50% 取现、+10% 利息赎回、被抵押不收租)。
- ✅ **GameConfig 参数化**:§4A 全部数值(28 格棋盘/起始资金/租金曲线/各计时/上限)做成可配置,建房载入,不写死。
- ✅ **唯一权威时钟**:主时钟与子交互时钟互斥(D-004),超时投递合成命令走引擎。
- ✅ **房间运行时(单写者)+ WebSocket 网关骨架** + **三口子 seam**(身份②/合规③/嵌入①,仅接口+no-op)。
- ✅ **§4B 回归测试**:29 条 vitest 用例全绿,覆盖 05-acceptance 的 20 条 seed(TC-TRADE/AUCTION/MORTGAGE)+ 回合主循环 + 单写者/时钟集成。

> 说明:§1 为 M1 首版交付清单;前端、重连、快照落盘、时长硬上限已在 M2/M3 补齐(见 §3 任务表与变更记录),当前测试 38 条全绿。

## 2. 明确不做(仍延后)
- 微信小程序端渲染(协议已端无关,留待接小程序 WebSocket 适配)。
- AI 托管"会玩"强度升级(首版被动规则已防卡场,二期增强)。
- 机会/命运卡的完整卡池与洗牌(首版用固定小卡堆循环)。
- 商业化/匹配/段位/排行榜/实名防沉迷(转 A 面向公众时再做,见 PRD §2 非目标)。

## 4. 里程碑
- M1(已达成):最高风险件 §4B 三子状态机打通 + 回归测试全绿 + 网关骨架可运行。
- M2(已达成):前端最小可玩页面 + 大厅/会话/重连 + 简单 AI 托管 → 端到端跑通完整一局(e2e 测试到结算)。客户可本地启动试玩。
- **M3(本轮已达成):闭合验收标的两个待跟踪项 —— ① 30 分钟时长硬上限接 wall-clock 强制结算(≤30 分钟收束 P0 达标);② 快照落盘 + 重启恢复。38 用例全绿。**
- 后续:试玩调参(GameConfig)、AI 托管"会玩"强度升级、测试工程师复验 ① + 补全 05-acceptance 其余 P0。

## 5. 依赖与阻塞
- 无阻塞。可转测试工程师**复验 ①**(8 人局 ≤30 分钟收束)并补全 05-acceptance 其余 P0(完整闭环/收束/重连资产不丢/防作弊抽检)。
- 已知限制(非阻断):从崩溃快照恢复时,时长硬上限按**完整 30 分钟重计**(未持久化剩余时间);快照为变更驱动的全量 JSON(熟人低并发够用,放量转 A 再换持久层)。

## 6. 代码位置 / 如何启动试玩
- 引擎:`src/engine/`;网关:`src/gateway/`(`room.ts` 单写者+时钟、`server.ts` WS+大厅+静态、`ai.ts` AI 托管、`seams.ts` 三口子);前端:`public/`。
- 启动:`npm install` → `npm run dev:server` → 浏览器开 `http://localhost:8080`,创建房间得房间码/链接,另开标签页或发链接给朋友输房间码进房,房主点开始即可对战。
- 测试:`npm test`(vitest,38 用例,含 §4B 回归 / e2e 完整一局 / 时长硬上限 / 快照恢复);类型检查:`npm run typecheck`。
