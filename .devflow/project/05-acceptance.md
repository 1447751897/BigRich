# 验收测试

> 本文当前为**架构师按 02-design §4B 回流的"交易/拍卖/抵押"长期回归用例 seed**(超时/掉线/同价/超额是首版最高风险件,必须长期回归)。
> 其余 P0(完整闭环、收束达标、断线重连、防作弊、AI 托管)的验收标准与用例,待 **测试工程师** 在本文基础上补全后,status 升 in_review/done。

## 1. 验收标准(对照 PRD §5)
- **交互不死锁**:交易/拍卖任一步超时都能自动作废/放弃并回到正常对局,不卡全场。
- **出价/成交原子性**:任何并发出价/交易下,不出现同价并存、超额(超过现金)成交、地产双主或资金不守恒。
- **计时隔离**:交易/拍卖子流程进行时,回合主时钟暂停;子流程结束才恢复——不出现"回合超时与交易超时同时触发"。
- **掉线兜底**:掉线玩家在交易中被动一律拒绝、在拍卖中直接放弃,且接管在 ≤1 回合内生效。
- (其余 §5 指标:完整闭环 / ≤30 分钟收束 / 重连资产不丢 / 服务端权威防作弊 —— 待测试工程师补)

| TC-TRADE-04 | 对方响应超时 | P 发起 → Q 20s 不操作 | 自动拒绝,交易作废,回 P 回合 | |
| TC-TRADE-05 | 还价后发起方超时 | P 发起 → Q 还价 → P 20s 不操作 | 自动拒绝,交易作废 | |

| TC-AUCTION-01 | 不买触发拍卖 | P 落无主地产选择不买 | 对全体在局玩家公开拍卖,挂起主时钟,自 P 下家起轮转 | |
| TC-AUCTION-02 | 正常成交 | 多人轮流加价,每轮 10s,连续一轮无人加价 | 成交给当前最高价者:扣现金+过户;退出拍卖恢复主时钟 | |

| TC-AUCTION-05 | 出价原子性·超额 | 出价 > 本人现金 | 服务端拒绝该出价(可在本轮 10s 内重出合法价) | |
| TC-AUCTION-06 | 单轮超时=放弃 | 轮到者 10s 不操作 | 视为放弃;若由此连续一轮无人加价则成交 | |
| TC-AUCTION-07 | 掉线者参与 | 拍卖中有玩家掉线 | 该玩家由托管直接放弃,不阻塞轮转 | |
| TC-AUCTION-08 | 成交资金守恒 | 成交后核对 | 买方现金减少额 = 成交价;无凭空增减;地产单一归属 | |

| TC-MORTGAGE-04 | 仅自回合 | 非自回合尝试抵押 | 服务端拒绝 | |
