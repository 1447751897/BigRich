## 1. 验收标准(对照 PRD §5 + §4B)
**P0 闭环 / 收束 / 联机健壮 / 防作弊(PRD §5):**
- **完整一局闭环**:2 人起可走通 开房→邀请→实时对战(买地/盖房/收租/机会卡/交易/拍卖/抵押)→ 破产淘汰或回合上限封顶 → 结算排名。
- **收束达标**:对局必然收束,不存在打不完的局;8 人局目标 **≤30 分钟**。
- **断线重连**:玩家掉线可重连回原局,**资产/进度不丢**,连接状态恢复;掉线期间由 AI 托管不卡全场。
- **防作弊(服务端权威)**:掷骰、资金、地产、交易/拍卖成交全部服务端裁决;**骰子绝不在客户端生成**;玩家只能以自己座位身份行动(issuer 服务端绑定)。
- **交互不死锁**(§4B):交易/拍卖任一步超时都能自动作废/放弃并回到正常对局。

**P0 §4B 交易/拍卖/抵押(架构师 seed):**
- **出价/成交原子性**:无同价并存、无超额(超现金)成交、无地产双主、资金守恒。
- **计时隔离**:子流程进行时回合主时钟暂停,结束才恢复。
- **掉线兜底**:掉线玩家交易中被动一律拒绝、拍卖中直接放弃,接管 ≤1 回合生效。

## 4. 已知问题 / 遗留(验收发现)
- **条件①(收束 · 阻断"≤30分钟"达标)**:`hardTimeLimitMin: 30` 已在 `config.ts`/`types.ts` 声明,但**引擎未接入 wall-clock 强制结算**。当前唯一生效的收束是**每人 20 回合上限**(`maxTurnsPerPlayer`)。回合上限保证"必然收束",但 8 人全程超时的极端情况(8×20×25s ≈ 66 分钟)**不能保证 ≤30 分钟**——这是 PRD §5 明列的 P0 指标尚未满足项。开发已将其列为下一步("接 wall-clock 留下一轮")。
- **条件②(重连健壮性 · 非阻断)**:断线重连在**进程内**已验证资产/进度不丢(TC-RECON-01);但**快照尚未落盘**(D-005,开发已注明"快照真正落盘留下一轮")。即:玩家掉线重连 OK,但**服务端进程重启/崩溃后无法恢复对局**。PRD §5 的"掉线重连"已满足,崩溃恢复属后续健壮性增强。
- **建议(非 P0 阻断)**:8 人满载并发的实时压测、去 IP 化(P1)文案/配色专项核对,建议在后续迭代补。
