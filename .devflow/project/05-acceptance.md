## 1. 验收标准(对照 PRD §5 + §4B)
**P0 闭环 / 收束 / 联机健壮 / 防作弊(PRD §5):**
- **完整一局闭环**:2 人起可走通 开房→邀请→实时对战(买地/盖房/收租/机会卡/交易/拍卖/抵押)→ 破产淘汰或封顶 → 结算排名。
- **收束达标**:对局必然收束,**8 人慢拖也能 ≤30 分钟**——回合上限(每人 20 回合)与 30 分钟时长硬上限**取先到先结算**。
- **断线重连**:玩家掉线可重连回原局,**资产/进度不丢**;**服务端重启/崩溃后亦可从快照恢复进行中房间**,旧 token 仍可重连。
- **防作弊(服务端权威)**:掷骰、资金、地产、交易/拍卖成交全部服务端裁决;**骰子绝不在客户端生成**;玩家只能以自己座位身份行动(issuer 服务端绑定)。
- **交互不死锁**(§4B):交易/拍卖任一步超时都能自动作废/放弃并回到正常对局。

**P0 §4B 交易/拍卖/抵押(架构师 seed):** 出价/成交原子性(无同价并存/超额/双主、资金守恒)、计时隔离(子流程进行时主时钟暂停)、掉线兜底(交易被动拒绝、拍卖放弃,≤1 回合接管)。

**前端表现层长期回归项(来自迭代 001-page-beautify,2026-05-30 回流):**
- **一屏自适应**:桌面(1366×768 / 1920×1080)与手机竖屏(如 390×844)下,棋盘 + 当前操作区一屏完整、无需滚动完成一回合。
- **位置可见 + 动画**:任意时刻一眼看出各玩家与自己所在格(自己高亮);掷骰有动画、**落定点数 = 服务端 `DiceRolled`**;音效可静音并记忆。
- **去 IP**:命名/文案自创,无受保护商标元素。
- **不回退既有流程**:解散房间、再来一局、骰子真随机(quick 001/002/003 + PR #15)行为保持;`npm test` 仍 **41/41 绿**、`npm run typecheck` 通过。

## 4. 已知问题 / 遗留(非阻断,转二期)
- **① 重启后时长上限按完整时长重挂**:从快照恢复进行中房间时,30 分钟硬上限定时器重新挂满 30 分钟(非剩余时长,见 `room.ts` 注释)。一局若恰好横跨一次服务端重启,wall-clock 可能略超 30 分钟。边缘情形、首版可接受;精确化方案(快照存开局时间戳、恢复按剩余时长重挂)转二期。
- **② AI 托管"会玩"强度升级**:首版为"防卡场"规则 AI(§4C),更强的决策 AI 转二期。
- 另:**8 人满载实时并发压测**未做专项(功能上限由 `joinSeat` room-full=8 守卫),建议后续迭代补。
